ARCHITECTURE DECISIONS

关键技术决策

原生 HarmonyOS使用 ArkTS、ArkUI 和 Stage 模型，不用 WebView 套壳承担主体验。确保多设备布局、接续、服务卡片和实况窗真正进入作品能力。
模块化单体优先业务域在代码层隔离，API 与 Worker 分进程，不提前拆微服务。降低竞赛阶段部署、调试和事务一致性成本，保留后续拆分空间。
PostgreSQL 统一数据关系数据、全文检索与 pgvector 共用一个可靠数据平台。P0/P1 不额外引入 Elasticsearch，减少运维面。
AI Provider 可替换VLM、文本模型、Embedding 和 OCR 均通过网关与结构化协议接入。支持成本控制、故障切换、模型评测与避免供应商锁定。
对象存储双实现本地与演示使用 MinIO，生产使用 OBS，业务只依赖统一存储接口。开发可复现，生产获得托管可靠性和生命周期能力。
人工确认优先关键 AI 输出先成为草稿或待确认结果，不直接改变业务责任。避免误判直接影响报名、组队、分工、审批和现场处置。
