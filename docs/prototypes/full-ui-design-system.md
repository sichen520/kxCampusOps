# CampusOps full UI prototype set

## Product direction

CampusOps is an evidence-driven campus action workspace. It helps students move from a public opportunity to feasibility judgement, team formation, task planning, and execution while preserving the source evidence behind each decision.

## Shared visual system

- Canvas: cool neutral `#F3F5F4`
- Work surfaces: white `#FFFFFF` and quiet alternate `#F7F8F7`
- Text: charcoal `#171C1A`, secondary `#626B67`
- Primary action: emerald `#087443`
- Evidence and source links: blue `#175CD3`
- Risk: amber `#A15C00` and red `#B42318`
- Borders: `#D7DDDA`, one-pixel hairlines
- Corners: 4-6 px, reserved for controls and bounded work surfaces
- Typography: compact Chinese sans-serif, strong hierarchy, no viewport-scaled display type

## Persistent desktop shell

- 248 px white left navigation with a green square `行` mark and `可行 CampusOps`
- Navigation: `首页`, `可行`, `知享`, `队伍`, `执行`, `我的`, and role-gated `管理`
- Compact white top workspace header with the current page title, calendar action, and signed-in avatar
- Workflow pages add a five-step progress line: `机会识别`, `可行性分析`, `方案规划`, `执行准备`, `报名提交`

## Signature

The signature element is the evidence chain: source links, judgement state, owner, deadline, and next action stay visually connected across tables, details, applications, and project tasks.

## Avoid

- Card soup, nested cards, and large rounded containers
- Purple, gradients, decorative blobs, glass effects, and marketing composition
- Oversized hero copy and generic AI explanations
- Excessive pills and badges
- Device frames, perspective mockups, watermarks, and unreadable placeholder text
